Cluster Usage Notes

  • Adding a user account
    If user has an account on the old cluster then use the uid defined in /etc/passwd.BAK on leo.cfr.nist.gov

    cuseradd -u uid user

    Set password, synchronize and add samba account:

    passwd user
    pwconv
    passsync

    Set up a samba account

    smbpasswd -a user

  • Powering down the cluster
    • su to root
    • Power off the compute nodes. Type:

      POWEROFF (all CAPS)
      you will be given one chance to change your mind. This script powers down nodes: node001->node032 .

    • Power off the head node. Type:

      poweroff (lowercase)

    • Turn off the switches
    • Turn off the UPS at the bottom of the cabinet

  • Powering up the cluster (reverse of power down instructions)
    • Power up the switches
    • Power up the UPS
    • Power up the head node. (wait until head node boots up before proceeding)
    • Power up the compute nodes

  • Setting clocks to the correct time

    Head node

    1. /etc/init.d/ntpd stop
    2. ntpdate "0.centos.pool.ntp.org"
      Type the above several times until the offset displayed is close to 0.
      Note: you may need to wait a few minutes after blaze boots up before this command will work.
    3. hwclock --systohc
    4. /etc/init.d/ntpd start

    Compute nodes

    1. fornodesp "/etc/init.d/ntpd stop"
    2. fornodesp "ntpdate blaze"
      type the above several times until the offset displayed is close to 0
    3. fornodesp "hwclock --systohc"
    4. fornodesp "/etc/init.d/ntpd start"

FDS Usage Notes

  • Running a serial FDS case
    qfds.sh path_to_fds casename.fds (user specified FDS)
    qfds.sh -r casename.fds (repository FDS)

  • Running a parallel FDS case
    qfds.sh -p nprocs path_to_fds casename.fds (user specified FDS)
    qfds.sh -r -p nprocs casename.fds (repository FDS)

  • Examining run queues
    qstat -a (all jobs)
    qstat -u user (all jobs by user)
